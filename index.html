02 ·The loop From any URL · to deployable HTML

Take any website.
Bring back the redesign.

Four sub-commands. Run them in order, or jump in at any point. Each step writes its own artifacts; the next step reads them.

stardust
Extract crawl + capture brand › Direct resolve direction, roll seed › Prototype 3 variants per page › Migrate ship deployable HTML
Extract

The current state, captured.

Crawl any URL. Capture the brand surface — palette, type, voice, motifs. The current state is described, not authored.

Direct + prototype

The seed picks the lane.

Roll a seed. Resolve a direction. Generate three variants for every page. Iterate in the browser.

Migrate

Static HTML, deployable.

Output every page as deployable static HTML. No build step, no framework lock-in. Drop it on any host that serves files.

03 ·The AI-facing brand Canonical · structured · curatable

The canonical brand,
for the AI-facing web.

Stardust answers a question your team will be asked more often: what is the canonical, AI-facing expression of this brand today? It produces the default experience — the reference point from which all variations are measured — in a form your marketing team can update and curate without developer intervention at every touchpoint.

Structured signal

Richer signals for AI to read.

When AI synthesizes your brand — Google AI Mode, Claude, search synth — it draws on available signals. Stardust output provides richer, more intentional signals than a static page designed for a pre-AI web.

Reproducible by date

Same seed, same brand.

Today's expression is locked. Tomorrow's seed yields a related-but-distinct variation. The default is always reproducible — yours alone, today only.

Curatable, not handed off

Marketing keeps the keys.

Three variant theses per page, each rendered, each rationalized. The brand team picks. Updates land without developer intervention at every touchpoint.

05 ·Scale to production One option · zero friction

Deploy anywhere. Scale with AEM.

Stardust is a generic redesign tool — drop the output on GitHub Pages, your CDN, or any host that serves files. When you need to manage experiences at scale (multiple markets, in-place authoring, drafts, translations, scheduled releases), AEM Edge Delivery is a frictionless route to production. The bridge is Snowflake, which converts the Stardust output into an EDS overlay page that's authorable in Document Authoring without losing the design.
