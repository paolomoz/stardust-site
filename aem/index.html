01 ·The shift From parts › pages

Web publishing changed under us.

From assembling pages out of shared parts → to generating unique pages that share the same rules.

The old deal

Reuse everything. Build nothing new.

Implementation partners shipped a library — Hero, Card Grid, Accordion, Tabs — and marketers composed pages from the shelf. Sites got fast by getting boring. It worked.

What broke it

The cost of uniqueness collapsed.

AI tools now generate bespoke, production-quality HTML — scroll-driven motion, cinematic transitions, three-week designs in an hour — from a prompt and a brand surface. Designers know how to drive them.

Now

The library was a workaround.

Every page can be its own page again — provided the brand keeps the rules. The library's job moves up a level: from shape to guardrails.

02 ·Why AEM Edge Delivery Beyond serving HTML · managing experiences

More than a renderer. An experience system.

AEM Edge Delivery serves document-authored static HTML at top-decile performance — but the deeper value is what comes with it: an experience platform for managing every market, every language, every campaign, while keeping the brand control non-negotiable. Stardust generates the artifact; AEM is where that artifact lives at scale.

Manage at scale

Every market, every campaign, one surface.

Authors edit in-place with Universal Editor. Drafts, scheduling, translations, A/B variants, market overrides — all in one place. Roll the same Stardust artifact across regions, products, and seasons without rebuilds.

Brand visibility

Optimized for the AI-facing web.

LLMs synthesize brands from what they can read. Edge Delivery's clean, semantic, structured markup — exactly what Stardust outputs — is what Google AI Mode, Claude, and ChatGPT extract correctly. Your brand survives the synth as you mean it.

Performance, by construction

Top-decile Core Web Vitals.

100/100 Lighthouse scores aren't aspirational on Edge Delivery — they're the default. Static HTML, edge-cached globally, no client-side framework rehydration. SEO and search ranking follow.

03 ·The bridge: Snowflake Stardust HTML › AEM EDS

Meet Snowflake.
From static HTML to a live AEM experience.

Snowflake is the open-source converter that turns a Stardust page into an AEM Edge Delivery experience. It uses an overlay pattern: the original DOM is preserved byte-for-byte while text and image content become authorable in Document Authoring. Header and footer drop into static repository fragments. Page CSS and any animation JavaScript ship per-template under the EDS code bus.

04 ·Ship your first page Two commands · URL to AEM

Two commands. No development in between.

From a live URL to a working AEM Edge Delivery experience. Stardust generates the redesign; Snowflake converts it into an EDS overlay page authorable in Document Authoring.

  1. Generate the redesign. /stardust:uplift <URL> — Stardust crawls the live site, captures the brand surface, and generates three differentiated variants in one run. Approve the one that fits.
  2. Convert and ship to AEM. /snowflake <FILE> to AEM — Snowflake wraps the chosen variant with the EDS overlay engine and pushes it into your AEM Edge Delivery project. Preview, then publish.
